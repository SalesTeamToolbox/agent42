    Class `UnifiedContextTool(Tool)`:
    - `requires = ["memory_store", "skill_loader", "workspace"]`
    - `__init__(self, memory_store=None, skill_loader=None, workspace="", effectiveness_store=None, **kwargs)`:

    - `parameters` property returns schema with:

      - `max_tokens` (integer, optional, default 4000) — "Max tokens in returned context"
      - `task_type` (string, optional) — "Override task type for effectiveness ranking (e.g., 'coding', 'debugging')"

    Method `async execute(self, topic="", scope="project", depth="quick", max_tokens=4000, task_type="", **kwargs)`:
    1. If not topic, return `ToolResult(output="", error="topic is required", success=False)`
    2. Infer task type: `inferred_type = task_type or _infer_task_type(topic)`
    3. Extract keywords: `keywords = _extract_keywords(topic)` (imported from context_assembler)
    4. Compute initial budgets dict from `_UCT_BUDGET_*` constants times `max_tokens`
    5. Determine if code symbols should be fetched: `fetch_code = inferred_type in _CODE_TASK_TYPES or not inferred_type`
    6. Run all sources concurrently with `asyncio.gather(..., return_exceptions=True)`:
       - `self._assembler.execute(topic=topic, scope=scope, depth=depth, max_tokens=max_tokens, **kwargs)` — base context
       - `self._fetch_code_symbols(topic, budgets["code"])` if `fetch_code` else a coroutine returning None
       - `self._fetch_gsd_state(keywords, budgets["gsd"])` — GSD workstream context
       - `self._fetch_effectiveness(inferred_type, budgets["effectiveness"])` — ranked tools
    7. Process results: for each result, if it's an Exception, log at WARNING and treat as None
    8. Extract base assembler output (result[0]): if it's a ToolResult, use its `.output`; if Exception, set to ""
    9. Collect additional sections from results[1:] (code symbols, GSD state, effectiveness) — skip None values
    10. Budget redistribution (D-14): count how many of the 3 additional sources returned None; redistribute their budget proportionally to sources that returned content, then re-truncate each section to its new expanded budget
    11. Assemble final output: base assembler output + "\n\n" + additional sections joined by "\n\n"
    12. Apply final `_truncate_to_budget(output, max_tokens)`
    13. Add footer with section count and token estimate
    14. Return `ToolResult(output=output, success=True)`

    Method `async _fetch_code_symbols(self, query: str, max_tokens: int) -> str | None`:
    - Per-call MCPConnection pattern (D-04, research Pattern 1):
    - `config = {"command": "uvx", "args": ["jcodemunch-mcp"]}`
    - `conn = MCPConnection("jcodemunch", config)`
    - In `try` block:
      - `await asyncio.wait_for(conn.connect(), timeout=3.0)`
      - Call `search_symbols` with `{"query": query, "repo": "local/agent42"}` — timeout 5.0s
      - If result is non-empty, also call `search_text` with same query — timeout 5.0s
      - Deduplicate between the two results using SHA256 on first 200 chars
      - Return `_truncate_to_budget(f"## Code Symbols\n\n{combined}", max_tokens)`
    - `except Exception as e`: log debug, return None
    - `finally`: disconnect conn (wrapped in try/except to prevent cleanup errors)

    Method `async _fetch_gsd_state(self, keywords: list[str], max_tokens: int) -> str | None`:
    - Per research Pattern 3 (D-07, D-08, D-09):
    - `workspace = Path(self._workspace)` if workspace else `Path(".")`
    - `state_files = list(workspace.glob(".planning/workstreams/*/STATE.md"))`
    - For each state_path, read with `aiofiles.open(state_path, "r")`
    - Parse YAML frontmatter (between `---` markers) with `yaml.safe_load()` wrapped in `try/except yaml.YAMLError`
    - Skip if `status` is `"Complete"` (case-insensitive)
    - Track most recently updated via `last_updated` field (handles multiple active workstreams per research Pitfall 3)
    - For the most recent active workstream:
      - Check keyword relevance: count overlap between `keywords` and words in `stopped_at` + workstream directory name
      - If overlap >= 1 keyword, include GSD context
      - Read ROADMAP.md from same workstream directory for current phase goal
      - Read active phase PLAN.md files (glob `phases/*/` and find open plans) for open task list
      - Format: `## GSD Workstream\n\n**Status:** {status}\n**Current:** {stopped_at}\n\n{phase_goal}\n\n**Open tasks:**\n{task_list}`
    - Return `_truncate_to_budget(section, max_tokens)` or None if no match

    Method `async _fetch_effectiveness(self, task_type: str, max_tokens: int) -> str | None`:
    - Per research Pattern 4 (D-10, D-12):
    - If `not self._effectiveness_store or not task_type`, return None
    - Call `self._effectiveness_store.get_recommendations(task_type=task_type, top_k=5)`
    - If empty, return None
    - Format each rec: `- **{tool_name}**: {rate}% success on {task_type} tasks ({invocations} calls, avg {avg_ms}ms)`
    - Return `_truncate_to_budget(f"## Effective Tools for {task_type}\n\n{content}", max_tokens)`
    - Wrap in `try/except Exception`, return None on failure
