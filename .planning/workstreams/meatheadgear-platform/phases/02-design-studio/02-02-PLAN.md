  <action>
    Modify `apps/meatheadgear/frontend/app.js` to add Fabric.js canvas management and upload handling.

    **State additions** -- Add these fields to the existing `state` object:
    - `fabricCanvas: null` -- the Fabric.js Canvas instance
    - `designImage: null` -- the currently placed FabricImage on canvas
    - `uploadedDesignUrl: null` -- data URL from file upload (DES-08)

    **Canvas initialization** -- Replace `initDesignStudio()` to call `initCanvas()` after session setup. New function `initCanvas()`:
    - Dispose previous canvas if `state.fabricCanvas` exists
    - Create: `state.fabricCanvas = new fabric.Canvas('design-canvas', { backgroundColor: '#151515', selection: true, preserveObjectStacking: true })`
    - Call `drawProductTemplate()` and `drawPrintAreaGuide()`

    **drawProductTemplate()**: Add a non-selectable `fabric.Rect` with fill '#2a2a2a', positioned at 10% from left, 5% from top, 80% width, 90% height, with rx/ry 10. Send to back.

    **drawPrintAreaGuide()**: Add a non-selectable `fabric.Rect` with transparent fill, stroke 'rgba(255, 32, 32, 0.3)', strokeDashArray [8, 4], positioned at 20% from left, 15% from top, 60% width, 50% height.

    **placeDesignOnCanvas(imageUrl)**: Async function that:
    - Removes previous `state.designImage` if exists
    - Loads image via `fabric.FabricImage.fromURL(imageUrl, { crossOrigin: 'anonymous' })` (per D-08 -- Fabric v6 class name)
    - Scales to fit print area (60% canvas width, 50% canvas height, capped at 1.0)
    - Centers on canvas (originX/originY: 'center')
    - Sets handle colors to #ff2020 (accent), cornerSize 10, transparentCorners false
    - Adds to canvas, sets as active object, stores as `state.designImage`
    - Shows preview-actions div

    **Canvas control functions:**
    - `resetCanvasDesign()`: Remove state.designImage, hide preview-actions
    - `centerDesignOnCanvas()`: Set designImage left/top to canvas center
    - `fitDesignToCanvas()`: Scale to fit 60%/50% of canvas, then center

    **exportCanvasAsPNG()**: Returns `state.fabricCanvas.toDataURL({ format: 'png', multiplier: 2 })` per D-10.

    **handleDesignUpload(event)** (DES-08):
    - Get file from event.target.files[0]
    - Validate type is in ['image/png', 'image/jpeg', 'image/svg+xml']
    - Validate size under 10MB
    - Use FileReader.readAsDataURL
    - On load, store as state.uploadedDesignUrl and call placeDesignOnCanvas(dataUrl)
    - Append system chat message: "Your uploaded design has been placed on the canvas."

    **Modify handleChatSubmit()**: After successful generation (`state.latestDesign = data`), replace the old static image preview code with:
    - `await placeDesignOnCanvas(data.image_url)`
    - Change chat message to: "Here's your design! Placed on canvas -- drag, resize, or rotate to position."
    - Remove the old code that created a static img element in preview-canvas

    **handleGenerateMockup()**: Export canvas PNG, show mockup modal with loading state, POST to `/api/design/mockup` with `{ design_id, canvas_data_url, product_id }`, display returned mockup image.

    **handleSaveDesign()**: POST to `/api/design/save` with `{ design_id, canvas_data_url }`, show success message in chat.

    **handleProceedToCheckout()**: Close mockup modal, alert "Checkout flow coming soon!" (Phase 3).

    **Modify newDesignSession()**: After clearing chat messages, call `initCanvas()` instead of manually creating a placeholder div. Reset state.latestDesign and state.uploadedDesignUrl to null.

    **Remove handleBuyDesign()** -- replaced by handleGenerateMockup + handleSaveDesign.
