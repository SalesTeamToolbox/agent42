**CRITICAL:** The task filter conditions must be appended AFTER the full forgotten/project filter assembly, not to the `conditions` list. The `conditions` list is only used in the fallback path (`forgotten_filter is None`).

After the existing filter assembly code (after the `project_filter` handling, around line 556), and before the `query_filter = ...` assignment:

```python
        # Task-type and task-id filtering (append to whatever filter was built)
        task_conditions = []
        if task_type_filter:
            task_conditions.append(FieldCondition(key="task_type", match=MatchValue(value=task_type_filter)))
        if task_id_filter:
            task_conditions.append(FieldCondition(key="task_id", match=MatchValue(value=task_id_filter)))

        if task_conditions:
            if forgotten_filter:
                forgotten_filter.must = (forgotten_filter.must or []) + task_conditions
            else:
                # Also add to conditions list for the fallback path
                conditions.extend(task_conditions)
```

This handles both code paths:
- When `forgotten_filter` exists (exclude_forgotten=True or project_filter set), task conditions are appended to `forgotten_filter.must`
- When `forgotten_filter` is None, task conditions are in the `conditions` list used by the fallback `Filter(must=conditions)` construction
