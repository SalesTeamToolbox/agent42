## Broken E2E Flows

### FLOW-01: File Save in Non-Default Workspace (CRITICAL)

- **Steps:** User edits file in workspace B → Ctrl+S → `ideFileSave()` → `POST /api/ide/file`
- **Breaks at:** `workspace_id` placed in JSON body → FastAPI reads it from query string (always `None`) → `_resolve_workspace(None)` → default workspace path
- **Impact:** File writes in workspace B silently go to workspace A's root. Data corruption risk.
- **Fix:** Add `workspace_id` to `IDEWriteRequest` model or send as query param

### FLOW-02: Search in Non-Default Workspace (MEDIUM)

- **Steps:** User searches in workspace B → `ideDoSearch()` → `GET /api/ide/search?q=...`
- **Breaks at:** `workspace_id` never appended to search URL → server uses default workspace
- **Impact:** Search results show files from wrong workspace
- **Fix:** Append `&workspace_id=_activeWorkspaceId` in `ideDoSearch`

### FLOW-03: Remove Workspace Unsaved Guard (LOW)

- **Steps:** User edits file in active workspace → clicks X on workspace tab
- **Breaks at:** `removeWorkspace` counts unsaved files from `_wsTabState` which may not reflect current `_ideTabs` modified flags
- **Impact:** Guard may skip confirmation dialog. User loses unsaved edits.
- **Fix:** Call `_saveCurrentWsState()` at the top of `removeWorkspace`
