**1. `read_hook_metadata(hook_path: str) -> list[dict]`**
Parse frontmatter comments from a hook `.py` file. Returns a list of registration dicts (one per `# hook_event:` line). Each dict has keys: `event` (str), `matcher` (str or None), `timeout` (int).

```python
def read_hook_metadata(hook_path: str) -> list[dict]:
    """Read hook_event/hook_matcher/hook_timeout frontmatter from a hook file.

    Returns list of registrations. Most hooks return 1 item.
    jcodemunch-reindex.py returns 2 (PostToolUse + Stop).
    """
    events = []
    matcher = None
    timeout = 30  # default
    with open(hook_path) as f:
        for line in f:
            stripped = line.strip()
            if not stripped.startswith("#"):
                break  # Stop at first non-comment, non-blank line
            if stripped == "":
                continue
            if stripped.startswith("# hook_event:"):
                events.append(stripped.split(":", 1)[1].strip())
            elif stripped.startswith("# hook_matcher:"):
                matcher = stripped.split(":", 1)[1].strip() or None
            elif stripped.startswith("# hook_timeout:"):
                try:
                    timeout = int(stripped.split(":", 1)[1].strip())
                except ValueError:
                    pass
    return [{"event": ev, "matcher": matcher, "timeout": timeout} for ev in events]
```

Note: blank comment lines (`#`) should not break parsing — only lines that don't start with `#` should stop the loop. Also, the shebang line `#!/usr/bin/env python3` starts with `#` so it will be iterated but won't match any key. This is correct.

**3. `register_hooks(project_dir: str) -> None`**
Read all `.py` files in `project_dir/.claude/hooks/`, call `read_hook_metadata()` on each, and merge into `project_dir/.claude/settings.json`.

- Load existing `settings.json` if it exists and parses; otherwise start with `{}`.
- Preserve the `permissions` key if present (never touch it).
- For `hooks` key: build the Claude Code hook registration structure.
- Group hooks by (event, matcher) tuple. Hooks with the same event and same matcher go in one registration block. Hooks with different matchers under the same event go in separate registration blocks.
- The registration structure per event is a list of registration blocks:
  ```json
  {
    "matcher": "Write|Edit",  // omit key if matcher is None
    "hooks": [
      {"type": "command", "command": "cd <project_dir> && python .claude/hooks/<script>", "timeout": <timeout>}
    ]
  }
  ```
- Use absolute `project_dir` in the command. Use forward slashes (Linux-only in Phase 1).
- Idempotency: before adding a hook command, check if a command containing the script filename already exists in that registration block's hooks list. If yes, skip.
- Write result with `json.dump(config, f, indent=2)` + trailing newline.

**4. `check_health(project_dir: str) -> list[dict]`**
Run 5 health probes, return list of `{"name": str, "healthy": bool, "detail": str, "fix": str, "level": "error"|"warn"}`.

Probes:
1. **MCP Server**: `subprocess.run([venv_python, os.path.join(project_dir, "mcp_server.py"), "--health"], capture_output=True, timeout=10)` — exit 0 = healthy.
   Fix: `"Check .venv and mcp_server.py imports"`. Level: `error`.
2. **jcodemunch**: `subprocess.run(["uvx", "jcodemunch-mcp", "--help"], capture_output=True, timeout=15)` — exit 0 = healthy. (Simple existence check, not full MCP handshake.)
   Fix: `"pip install uv && uvx jcodemunch-mcp"`. Level: `error`.
3. **Qdrant**: HTTP GET `http://localhost:6333/healthz` using `urllib.request.urlopen()` (stdlib, no deps). HTTP 200 = healthy.
   Fix: `"Start Qdrant: qdrant or docker run qdrant/qdrant"`. Level: `warn`.
4. **Redis**: `subprocess.run(["redis-cli", "ping"], capture_output=True, timeout=5)` — stdout contains `PONG` = healthy.
   Fix: `"Start Redis: sudo systemctl start redis"`. Level: `warn`.
5. **Claude Code CLI**: `subprocess.run(["claude", "--version"], capture_output=True, timeout=5)` — exit 0 = healthy.
   Fix: `"Install Claude Code: https://docs.anthropic.com/claude-code"`. Level: `error`.

Each probe must be wrapped in try/except to catch `FileNotFoundError`, `subprocess.TimeoutExpired`, `urllib.error.URLError`, etc. On any exception, the service is unhealthy with detail from the exception message.

**5. `print_health_report(results: list[dict]) -> None`**

- Pass: `\033[0;32m[✓]\033[0m {name}: healthy`

- Fail (warn): `\033[1;33m[✗]\033[0m {name}: {detail} → Fix: {fix}`
- Summary: `\nSetup complete. {X}/5 services healthy.`

```python
def main():
    """Entry point — parse transport argument and start server."""
    logging.basicConfig(
        level=logging.INFO,
        format="%(asctime)s [%(levelname)s] %(name)s: %(message)s",
        stream=sys.stderr,
    )

    # Health check mode — verify imports and config, then exit
    if "--health" in sys.argv:
        try:
            from core.config import Settings
            settings = Settings.from_env()
            # Verify critical imports work
            from tools.base import Tool, ToolResult
            from agents.agent import Agent
            print("healthy", file=sys.stderr)
            sys.exit(0)
        except Exception as e:
            print(f"unhealthy: {e}", file=sys.stderr)
            sys.exit(1)

    # ... rest of existing main() unchanged
```

2. Imports `Tool`, `ToolResult`, `Agent` — catches broken dependencies

4. Exits 0 on success, 1 on any exception
