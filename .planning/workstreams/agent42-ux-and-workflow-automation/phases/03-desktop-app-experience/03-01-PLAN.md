Create `scripts/generate-icons.py` — a Python script that converts the SVG favicon to PNG icons at multiple sizes (per D-07, D-08).

The script must:
1. Import `cairosvg` for SVG-to-PNG conversion (high-quality vector rendering, no pixelation at large sizes).
2. Accept an optional `--svg` argument defaulting to `dashboard/frontend/dist/assets/agent42-favicon.svg`.
3. Accept an optional `--output-dir` argument defaulting to `dashboard/frontend/dist/assets/icons/`.
4. Create the output directory if it does not exist (`os.makedirs(output_dir, exist_ok=True)`).
5. Generate these files using `cairosvg.svg2png()`:
   - `icon-192.png` — 192x192 pixels (`output_width=192, output_height=192`)
   - `icon-512.png` — 512x512 pixels (`output_width=512, output_height=512`)
   - `apple-touch-icon-180.png` — 180x180 pixels (`output_width=180, output_height=180`)
6. Print each generated file path to stdout.
7. Add a shebang `#!/usr/bin/env python3` and `if __name__ == "__main__":` guard.

After creating the script, run it to generate the actual PNG files:
```bash
pip install cairosvg
python scripts/generate-icons.py
```

Per D-09, the generated PNGs are committed to the repo (not gitignored). Verify all three files exist and are non-zero bytes.

Note: cairosvg depends on Cairo graphics library. On Windows, it may need `pip install cairosvg` which bundles the DLLs. If cairosvg fails, fall back to Pillow with svglib+reportlab:
```python
from svglib.svglib import svg2rlg
from reportlab.graphics import renderPM
```
