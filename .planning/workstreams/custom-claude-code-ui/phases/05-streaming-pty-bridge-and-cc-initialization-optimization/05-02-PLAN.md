**Step 3: Create read function based on mode**

Define `_read_stdout` as one of three implementations depending on `use_cc_pty` and platform:

Windows PTY (uses `cc_pty_process.readline` via `run_in_executor`):
- Calls `cc_pty_process.readline` in executor (blocks until newline)
- Strips ANSI with `_ANSI_ESCAPE.sub("", line)`
- Parses JSON, calls `_parse_cc_event`, sends envelopes
- Catches `EOFError` to break loop
- Loop condition: `cc_pty_process.isalive()`

Unix PTY (uses `select` + `os.read` with line accumulation buffer):
- `_cc_line_buf = ""` accumulation variable
- `_read_cc_chunk()` helper: uses `select.select([cc_master_fd], [], [], 0.1)`, returns decoded string
- Main loop: accumulate chunks, split on `"\n"`, strip ANSI, parse JSON
- Loop condition: `cc_proc.poll() is None`
- Sleep 0.05s when no data available

PIPE fallback (unchanged from current implementation):
- `async for raw_line in cc_proc.stdout`
- Decode, strip, parse JSON, send envelopes
