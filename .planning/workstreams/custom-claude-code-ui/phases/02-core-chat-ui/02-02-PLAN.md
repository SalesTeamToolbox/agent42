The current cc_chat_ws (from Plan 01-02) has this sequential structure in the per-message loop:
```python
read_task = _asyncio.create_task(_read_stdout())
try:
    await read_task          # BLOCKS here — cannot receive WS messages while subprocess runs
except Exception:
    pass
finally:
    read_task.cancel()
    if proc.returncode is None:
        proc.terminate()
```

Replace the blocking `await read_task` with this asyncio.wait() concurrent pattern:
```python
# Create concurrent awaitables — receive_task watches for incoming WS messages
async def _receive_msg():
    return await websocket.receive_text()

receive_task = _asyncio.create_task(_receive_msg())
try:
    while True:
        done, pending = await _asyncio.wait(
            {read_task, receive_task},
            return_when=_asyncio.FIRST_COMPLETED
        )
        if receive_task in done:
            # A WS message arrived during generation
            try:
                inner = _json.loads(receive_task.result())
            except Exception:
                inner = {}
            if inner.get("type") == "stop":
                # Cancel the subprocess read — stop generation
                read_task.cancel()
                if proc.returncode is None:
                    proc.terminate()
                await websocket.send_json({"type": "turn_complete", "data": {
                    "session_id": session_state.get("cc_session_id"),
                    "cost_usd": None, "input_tokens": 0, "output_tokens": 0
                }})
                receive_task = None
                break
            # Not a stop message — re-create receive_task for next message; read_task is still running
            receive_task = _asyncio.create_task(_receive_msg())
            if read_task in done:
                break  # subprocess also finished — turn complete
        else:
            # read_task finished (subprocess exited) — turn complete
            for t in pending:
                t.cancel()
            break
finally:
    if receive_task and not receive_task.done():
        receive_task.cancel()
    read_task.cancel()
    if proc.returncode is None:
        proc.terminate()
```

Existing aliases in create_app scope (do NOT redefine): _asyncio, _json, workspace.
