**Change A — Deduplicate assistant event (UX-03 root cause fix):**

The duplicate answer bug occurs because CC emits BOTH incremental `stream_event` `text_delta` events AND a final `assistant` event containing the full response text. `_parse_cc_event` converts both into `text_delta` WS envelopes, causing the frontend to render the text twice.

Fix: Add a `has_streamed_text` boolean to `session_state`. Set it to `True` when the first `stream_event` `text_delta` envelope is created. In the `assistant` event handler (line 2050), check `session_state.get("has_streamed_text")` — if `True`, skip emitting the `text_delta` envelope (the text was already streamed incrementally). Reset `has_streamed_text` to `False` in the `result` event handler (line 2060) so it resets per-turn.

Specific edits:
1. In `cc_chat_ws` where `session_state` is initialized (around line 2118-2125), add: `session_state["has_streamed_text"] = False`
2. In `_parse_cc_event`, in the `stream_event` handler, inside the `text_delta` case (around line 1992-1998), after appending the text_delta envelope, add: `session_state["has_streamed_text"] = True`
3. In `_parse_cc_event`, in the `assistant` handler (around line 2050), wrap the existing text extraction loop in: `if not session_state.get("has_streamed_text"):`
4. In `_parse_cc_event`, in the `result` handler (around line 2060), add: `session_state["has_streamed_text"] = False`

**Change A — Frontend dedup guard (belt-and-suspenders for UX-03):**

Even though the backend fix prevents duplicates, add a client-side guard as defense-in-depth. In `ccMakeWsHandler`, track a `turnTextHash` on the tab object. When `turn_complete` fires and renders the final markdown, compute a simple hash of `finalBuf`. Store it as `tab._lastTurnHash`. On the next `turn_complete`, if the hash matches and the content is identical, skip rendering. This catches any edge case where the same content arrives twice due to reconnect or retry.

Specifically:
1. In `ideOpenCCChat` where tab object is created (around line 4701), add property: `_lastTurnHash: ""`
2. In `ccMakeWsHandler` turn_complete handler (around line 4321-4343), before rendering the final markdown:
```javascript
var contentHash = finalBuf.length + ":" + finalBuf.slice(0, 100) + finalBuf.slice(-100);
if (contentHash === tab._lastTurnHash && finalEl) {
  // Duplicate turn — remove the duplicate bubble
  var dupWrapper = finalEl.closest(".chat-msg");
  if (dupWrapper) dupWrapper.remove();
  tab.streamMsgEl = null;
  tab.streamBuffer = "";
  tab.toolCards = {};
  ccSetSendingState(tab, false);
  return;
}
tab._lastTurnHash = contentHash;
```
